# Product Backlog

The backlog is based on use cases and divided into three planned sprints. Priority reflects business/gameplay value, while difficulty was estimated using Fibonacci story points.

## Sprint 1 - Core gameplay loop

| ID | Use case | Priority | Story points |
|---|---|---:|---:|
| PU001 | Scan QR code | Very high | 5 |
| PU002 | Collect item | High | 3 |
| PU003 | Execute action in the game world | Very high | 8 |
| PU004 | View character inventory | Medium | 2 |
| PU007 | Resolve action effect | High | 8 |

## Sprint 2 - Organizer configuration and monitoring

| ID | Use case | Priority | Story points |
|---|---|---:|---:|
| PU005 | Register presence in game area | High | 5 |
| PU006 | Define QR code / sensor meaning | High | 5 |
| PU009 | Monitor player resources | High | 5 |
| PU008 | View game event history | Medium | 3 |

## Sprint 3 - Extended game management

| ID | Use case | Priority | Story points |
|---|---|---:|---:|
| PU010 | Manage game terrain / rooms | Medium | 5 |
| PU011 | Define character and item profiles | Medium | 5 |
| PU012 | Review game timeline | Low | 3 |
