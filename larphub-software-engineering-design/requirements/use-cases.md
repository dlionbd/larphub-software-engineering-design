# Use Cases

The project focuses on the core gameplay loop and selected organizer functions.

## Actors

### Player
A participant using the mobile application during a LARP event.

### Organizer / Game Master
A person managing the event through a web application.

## Player use cases

| ID | Use case | Description |
|---|---|---|
| PU001 | Scan QR code | Player scans a QR code to identify a game object. |
| PU002 | Collect item | Player collects an item if inventory capacity allows it. |
| PU003 | Execute action in the game world | Player performs an available action at an interaction point. |
| PU004 | View character inventory | Player checks character inventory and available items. |
| PU005 | Register presence in game area | Player confirms presence in a defined game area. |

## Organizer use cases

| ID | Use case | Description |
|---|---|---|
| PU006 | Define QR code / sensor meaning | Organizer configures what a QR code or sensor represents. |
| PU007 | Resolve action effect | Organizer reviews and resolves effects triggered by player actions. |
| PU008 | View game event history | Organizer reviews the history of game events. |
| PU009 | Monitor player resources | Organizer monitors player and character resources during the event. |
